# DHT11、DHT22、AHT20 传感器开发规则

用途：指导 Embex 为温湿度传感器生成代码和诊断接线问题。

## DHT11 / DHT22

DHT11/DHT22 使用单总线数据协议，常见接线：

- VCC
- GND
- DATA

开发注意：

- DATA 通常需要上拉电阻，很多模块板已自带上拉。
- 读取间隔不能过短，DHT11 通常建议 1 秒以上。
- 初始化成功不代表读取成功，必须检查读数是否为 NaN 或错误码。
- GPIO 选择应避开明确的启动脚、USB 引脚和板卡文档标记的高风险脚。
- 普通 GPIO 上的 DHT11 读取失败时，应优先检查接线顺序、供电、电平、上拉电阻和代码中的 GPIO 编号。

## 项目经验修正

本项目曾出现过“接入 DHT11 后烧录异常”的现象。后续确认原因是当时接线错误，并非 GPIO11 或 DHT11 本身导致烧录失败。

正确知识应记录为：

- 这是一个误接线导致下载/烧录异常的历史案例。
- 不应把 GPIO11 永久标记为 DHT11 禁用引脚。
- 不应在没有烧录失败日志的情况下主动报告“GPIO11 DHT11 会导致烧录失败”；正确说法是“若接入新外设后出现烧录异常，应检查接线是否错误”。
- 只有当当前确实出现烧录失败、下载握手异常、`Failed to connect`、`Invalid head of packet` 等证据时，才把“新接外设或接线错误影响下载模式”作为排查方向。

## AHT20

AHT20 使用 I2C 通信，常见接线：

- VCC
- GND
- SDA
- SCL

开发注意：

- I2C 地址通常为 `0x38`。
- 初始化失败时应扫描 I2C 总线并输出扫描结果。
- 如果 OLED 也是 I2C，可共享 SDA/SCL，但必须确认地址不冲突。
- 如果 OLED 是 SPI，则 AHT20 的 I2C SCL/SDA 不应和 SPI 信号线混淆。

## 验收标准

- 串口输出传感器初始化结果。
- 读取成功时输出温度、湿度和单位。
- 读取失败时输出错误原因，例如未响应、地址错误、读数 NaN、总线冲突。
