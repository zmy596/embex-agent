# 项目案例：环境监测与声光显示终端

用途：给 Embex 的外设组合 Skill 提供可检索的小项目模板。

## 适用外设

- ESP32-C3 / ESP32-S3 / ESP32 / ESP8266 任一支持 Arduino 的板卡。
- OLED：显示状态、温湿度或错误信息。
- DHT11 / DHT22 / AHT20：采集温湿度。
- LED：运行状态或告警提示。
- 无源蜂鸣器：阈值告警或启动提示。

## 功能设计

1. 上电后串口输出 `[BOOT]`。
2. 初始化传感器、OLED、LED、蜂鸣器。
3. OLED 显示项目名称和当前状态。
4. 周期读取温湿度。
5. 串口输出结构化读数。
6. LED 以不同模式表示正常/告警。
7. 蜂鸣器在告警时短鸣。
8. loop 输出 heartbeat，便于判断固件仍在运行。

## 验收标准

- PlatformIO 编译成功。
- 有串口时烧录成功。
- 串口包含 `[BOOT]`、`[PIN]`、`[APP]`、`[HEARTBEAT]`。
- OLED 显示项目名或传感器读数。
- LED 行为符合设计。
- 传感器读取失败时有明确错误日志。
- 若蜂鸣器启用，发声行为与日志一致。

## 失败排查

- OLED 不亮：检查协议、控制器、RES/DC/CLK/MOSI/SDA/SCL。
- 传感器无读数：检查引脚、上拉、电源、读取间隔。
- 烧录失败：先断开 DHT11 等外设，尤其排查启动相关 GPIO。
- 重启循环：拆分初始化流程，定位卡死外设。
